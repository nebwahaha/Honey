# HoneyBlock

Cowrie honeypot with a one-click dashboard and IP blocking interface.

## Quick Start

```bash
sudo bash honeyblock-installer.run
```

See [TUTORIAL.md](TUTORIAL.md) for the full step-by-step guide.

## Development

### Prerequisites

- Python 3.10+
- Node.js 18+
- npm

### Setup

```bash
# Install backend dependencies
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Install frontend dependencies
cd ../frontend
npm install
```

### Running locally

Open two terminals:

```bash
# Terminal 1: Flask API
cd backend
source .venv/bin/activate
python app.py
```

```bash
# Terminal 2: Vite dev server (hot reload)
cd frontend
npm run dev
```

Open `http://localhost:5173` вЂ” Vite proxies API requests to Flask automatically.

### Making changes and seeing them live

#### Frontend changes (React/TypeScript)

If you're running `npm run dev`, changes appear instantly (hot reload). No extra steps.

To update the **installed** version (the one at `http://localhost:5000`):

```bash
# 1. Rebuild the frontend
cd frontend
npm run build

# 2. Copy to the installed location
sudo rm -rf /opt/honeyblock/backend/static
sudo cp -r ../backend/static /opt/honeyblock/backend/static

# 3. Restart the service
sudo systemctl restart honeyblock
```

#### Backend changes (Python)

```bash
# 1. Copy the changed file(s)
sudo cp backend/app.py /opt/honeyblock/backend/app.py

# 2. Restart the service
sudo systemctl restart honeyblock
```

For log_watcher.py changes:

```bash
sudo cp backend/log_watcher.py /opt/honeyblock/backend/log_watcher.py
sudo systemctl restart honeyblock-watcher
```

#### Quick one-liner to update everything

```bash
# Rebuild frontend, copy all files, restart all services
cd frontend && npm run build && cd .. && \
sudo rm -rf /opt/honeyblock/backend/static && \
sudo cp -r backend/ /opt/honeyblock/backend && \
sudo systemctl restart honeyblock honeyblock-watcher
```

### Building the installer

```bash
bash build.sh
```

This builds the frontend, packages everything into `honeyblock-installer.run`.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/stats` | Dashboard statistics |
| GET | `/api/attempts?page=1&limit=50` | Paginated attack log |
| GET | `/api/attackers` | Grouped attacker summary |
| GET | `/api/blocked` | List of blocked IPs |
| POST | `/api/block` | Block an IP `{"ip": "x.x.x.x"}` |
| POST | `/api/unblock` | Unblock an IP `{"ip": "x.x.x.x"}` |

## Services

| Service | Command | Purpose |
|---------|---------|---------|
| `cowrie` | `sudo systemctl status cowrie` | SSH honeypot on port 2222 |
| `honeyblock` | `sudo systemctl status honeyblock` | Dashboard API on port 5000 |
| `honeyblock-watcher` | `sudo systemctl status honeyblock-watcher` | Log ingestion daemon |


sudo systemctl stop cowrie honeyblock honeyblock-watcher
sudo systemctl disable cowrie honeyblock honeyblock-watcher
sudo rm /etc/systemd/system/cowrie.service /etc/systemd/system/honeyblock.service /etc/systemd/system/honeyblock-watcher.service
sudo systemctl daemon-reload
sudo rm -rf /opt/honeyblock
sudo userdel -r cowrie

sudo bash honeyblock-installer.run
