# HoneyBlock v2.0.1

A honeypot monitoring system that deploys a Cowrie SSH/Telnet honeypot, logs attacker activity to a SQLite database, and provides a real-time web dashboard with IP blocking via iptables.

![Dashboard](https://img.shields.io/badge/Dashboard-React%20%2B%20TypeScript-blue)
![Backend](https://img.shields.io/badge/Backend-Flask%20%2B%20SQLite-green)
![Platform](https://img.shields.io/badge/Platform-Ubuntu%2020.04%2B-orange)
![License](https://img.shields.io/badge/License-MIT-yellow)

## Features

- **Real-time Monitoring Dashboard** — Live feed of raw Cowrie logs, top attacker bar chart, and world map showing attack origins
- **IP Blocking** — One-click block/unblock attacker IPs using iptables firewall rules
- **3-Table Database** — Tracks attacker sessions, attacker profiles with geolocation, and block history
- **One-Command Installer** — Single `.run` file installs everything on a fresh Ubuntu machine
- **Desktop Shortcut** — Toggle HoneyBlock on/off from your Ubuntu desktop
- **Auto-Start** — Optional boot-on-startup for all services

## How It Works

```
Attacker tries SSH/Telnet (port 2222)
        |
        v
   [ Cowrie Honeypot ]  --logs-->  cowrie.json
                                       |
                                       v
                               [ Log Watcher ]  --parses-->  SQLite DB
                                                                |
                                                                v
                                                         [ Flask API ]
                                                                |
                                                                v
                                                       [ Web Dashboard ]
                                                       (localhost:5000)
```

## Quick Start

```bash
# Download the installer
wget https://github.com/nebwahaha/honeyblock/releases/download/v2.0.1/honeyblock-installer.run

# Run it
sudo bash honeyblock-installer.run
```

Open `http://localhost:5000` to access the dashboard.

See [TUTORIAL.md](TUTORIAL.md) for the full step-by-step guide.

## Dashboard Pages

| Page | Description |
|------|-------------|
| **Dashboard** | Stat cards (Total Attacks, Unique IPs, Blocked IPs, Active Sessions), Top 5 Attacker IPs bar chart, World map with attack locations, Live raw Cowrie log feed |
| **Blocking** | Detected attackers table with block/unblock buttons, Block history log |
| **Configurations** | Start/Stop Cowrie honeypot, Enable/Disable auto-start on boot |

## Database Schema

| Table | Description |
|-------|-------------|
| `attacker_session` | Every Cowrie event (connect, login, command, disconnect) with IP, credentials, commands, timestamps |
| `attacker` | Unique attacker profiles with first/last detection, country, block status |
| `blocklist` | Block history with dates, expiration, active status |

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/stats` | Dashboard statistics |
| GET | `/api/attempts?page=1&limit=50` | Paginated session log |
| GET | `/api/attackers` | Attacker profiles |
| GET | `/api/blocked` | Block history |
| GET | `/api/logs?limit=100` | Raw Cowrie log entries |
| POST | `/api/block` | Block an IP `{"ip": "x.x.x.x"}` |
| POST | `/api/unblock` | Unblock an IP `{"ip": "x.x.x.x"}` |
| GET | `/api/cowrie/status` | Cowrie running status |
| POST | `/api/cowrie/toggle` | Start/stop Cowrie |
| GET | `/api/autostart/status` | Auto-start status |
| POST | `/api/autostart/toggle` | Enable/disable auto-start |

## Services

| Service | Command | Purpose |
|---------|---------|---------|
| `cowrie` | `sudo systemctl status cowrie` | SSH/Telnet honeypot on port 2222 |
| `honeyblock` | `sudo systemctl status honeyblock` | Dashboard API on port 5000 |
| `honeyblock-watcher` | `sudo systemctl status honeyblock-watcher` | Log ingestion daemon |

## Development

### Prerequisites

- Python 3.10+
- Node.js 18+
- npm

### Setup

```bash
# Install backend dependencies
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Install frontend dependencies
cd ../frontend
npm install
```

### Running locally

```bash
# Terminal 1: Flask API
cd backend && source .venv/bin/activate && python app.py

# Terminal 2: Vite dev server (hot reload)
cd frontend && npm run dev
```

Open `http://localhost:5173` — Vite proxies API requests to Flask automatically.

### Building the installer

```bash
bash build.sh
```

## Uninstall

```bash
sudo systemctl stop cowrie honeyblock honeyblock-watcher && \
sudo systemctl disable cowrie honeyblock honeyblock-watcher 2>/dev/null; \
sudo rm -f /etc/systemd/system/cowrie.service /etc/systemd/system/honeyblock.service /etc/systemd/system/honeyblock-watcher.service && \
sudo systemctl daemon-reload && \
sudo rm -rf /opt/honeyblock && \
sudo userdel -r cowrie && \
sudo rm -f /usr/share/polkit-1/actions/com.honeyblock.ctl.policy && \
rm -f ~/Desktop/HoneyBlock.desktop
```
